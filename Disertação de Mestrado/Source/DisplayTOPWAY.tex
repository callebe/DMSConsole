A utilização de um ecrã sensível ao toque, ou \textit{Display touchscreen}, no projeto do console de indicadores de destino tem como objetivo tornar a interface do usuário mais versátil e interativa. Utilizando um \textit{Display touchscreen} é possível criar um conjunto de regras que delimitam certas zonas da superfície do \textit{Display}, que ao serem tocadas pelo usuário ativam o envio de mensagens ao dispositivo mestre, informando a ocorrência do evento. Utilizando este recurso é possível criar botões virtuais no \textit{Display} que desencadeiam eventos no dispositivo mestre, o qual de acordo com a sua programação responde ao \textit{display} com requisições que podem mudar os aspectos visuais do \textit{Frame} exibido. Através da dinâmica de botões virtuais e da mudanças de \textit{frames}  torna-se prático a criação de uma interface de usuário que se molda  criando botões e recursos visuais de acordo com a necessidade das requisições do usuário.  O TOPWAY HKT080ATA-C, apresentado na Figura (\ref{fig:DisplayTOPWAY}) é o modelo de \textit{Display touchscreen} disponibilizado pela DMS, para a realização deste trabalho.

 O  TOPWAY HKT080ATA-C é um modulo \textit{Smart display} de 7 polegadas TFT LCD (\textit{thin film transistor liquid crystal display}), o qual possui uma interface de comunicação UART-RS232. Por ser um módulo \textit{Smart} este \textit{display} vem equipado com um microcontrolador que gerencia os arquivos de imagens, armazenados na memória RAM também presente no módulo, com os quais ele constrói os \textit{frams} a serem exibidos.  Através da comunicação UART o dispositivo mestre configura e controla o microcontrolador.  
 
 O fabricante do TOPWAY HKT080ATA-C disponibiliza um programa de edição gráfica de interface, o \textit{TOPWAY TML Graphics Editor}, visto na Figura (\ref{fig:TMLGraphicsEditor}). Este editor cria o conjunto de dados a serem exportados, via conexão USB, para dentro da memória RAM do módulo.

Por meio desta aplicação é possível instanciar elementos gráficos, configurar comportamentos responsivos, alocar variáveis que armazenam eventos e configurar gatilhos para transmitir via UART-RS232 as informações sobre eventos. Ainda é possível designar áreas de texto na interface, tais áreas são preenchidas com informações que podem ser enviadas de um dispositivo mestre para o \textit{display} via UART. Assim basta criar os elementos gráficos da interface e suas ações, através do \textit{TOPWAY TML Graphics Editor}, e gerenciar a interface usando o dispositivo mestre. O dispositivo mestre utilizado neste trabalho é o Raspberry Pi - Zero W.
